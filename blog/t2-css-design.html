Position those elements!

a crash course on CSS positioning.

March 29, 2015

OH, CSS positiong. How you have caused me so many headaches this week. Figuring out which position property to use and when, can be a bit tricky. So I'm going to lay out all that I know - here. With the hopes that it will help you, and me, get a handle on this sometimes complicated property.

BUT, Before we get to positioning, we should talk about what CSS is, exactly. CSS stands for Cascading Style Sheets. Cascading Style Sheets contain special code that makes your website pretty. That's right, if HTML builds the structure of your page; then CSS designs it. The set up is simple. You start by selecting the element in the HTML that you want to manipulate. Then, when you decide what you want to do to that element, you choose the appropriate property. Finally, you plug in the value that suits your design needs. With CSS, you can change colors, fonts, sizes and even an element's position on the page (which is what we will be talking about today.) For our examples and explanations below we will be selecting the <main> element to position with CSS.

First let's discuss the static property value. Stactic positioning is the default position an element has on the page. It's the position the element has in the normal flow of the HTML document. You cannot use the top, right, bottom, left or z-index properties on this property value because, you don't need it. You will probably never see this property in a style sheet, because it basically tells elements to, "Stay in your natural flow!".

Next the relative property value, which can be a bit tricky. At least for me it was. Essentially, when you relatively position an element, you can manipulate it's location on the page - without actually changing the flow of that page. In other words, you can move the element, but the spot from which it originated, will still be there. The elements around this relatively positioned element act like it's still in it's normal spot. With this value, you can use properties like, top, right, bottom and left to determine the exact place on the page you want to stick your element.

main { position: relative; top: 120px; left: 300px; }

And then the plot thickens. Now we come to the absolute position. Which is similar to the relative position but different because when you absolutely position an element, it is moved from the flow of the page entirely. It's treated as if it where never even in the page all together. With this value, you can use the top, right, bottom and left properties to determine the exact location on the page, however, it's position will be relative to it's parent element. For example, if the <main> element is nested inside of a <div> element, then it will be positioned on the page according to the <div>'s location. In order for this to work, the parent element (or the <div> in our example) has to also have a position property with a value of relative, absolute or fixed (not static). Otherwise, it will position itself according to the overarching <html> element.

main { position: absolute; bottom: 350px; z-index: -250px; }

Last, but not least: the fixed position. Like the absolute position, the element is removed from the normal flow of the document. All the other elements around it act like it's not even there. The difference here is, the fixed value, fixes an element on the page according to the browser window. So even if the user scrolls down, the fixed element will still be there. This is good to use for social media icons, or menus that you want your user to always see.

main { position: fixed; top: 400px; right: 200px; }
