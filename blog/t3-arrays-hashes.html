Hashes vs. Arrays

the ultimate Ruby object-holding showdown

April 05, 2015

This week was all Ruby everything. Meaning, I got my first dose of the magical programming language. I learned two efficient ways to hold collections of information in Ruby. Arrays and Hashes. Today's conversation is all about that. We'll discuss the differences between each, and talk a little bit about when each should be used..

Arrays and Hashes are just ways to organize information in your programs. Each piece of information in array or hash has a key and a value. You can access that piece of information in your array or hash by it's key. Herein lies one of the fundamental differences of hashes and arrays. Let's jump right in:

Arrays

Arrays are ordered lists of information surrounded by square brackets. They can hold any type of data - strings, numbers or even booleans. You can even put an array IN an array. They are ordered because they indexed by their place in the list. Therefore, their key, or how you access that bit of information, can be 0, 1, 2, 3 etc. For example, index of the letter 'a' in the example below is 0, and 'b' is 1, 'c' is 2 and so on.

alphabet = [ 'a', 'b', 'c', 'd', 'e', 'f', 'g']
puts alphabet[0]
output-> a

Hashes

Hashes are the more complicated of the two. Hashes store bits of information organized how you want. Instead of index integers as keys, you can create your own using strings or numbers and assign them a value using this fancy symbol => (just an equals sign and a greater than symbol mushed together). There are multiple ways to create Hashes, here are a couple:

Literal Hash - good for values you probably won't change

my_info = {
"name" => "Nikki",
"age" => 28,
"city" => "Richmond",
"state" => "California"
}

Hash.new (applying the new method creates a new and empty Hash.)

Once created you can stick stuff by assigning the key/value pair as shown below

my_info = Hash.new
my_info["age"] = 28

These are just two ways to create Hashes, there are a couple more, but today we'll just address the basics. In this the example above, the keys are name, age, city, and state. They can be used to figure out the values in they represent. If you try to access a key that has no value, you will get nil value, in other words nothing. Check out the example below:

puts my_info["name"]
output -> Nikki

When to use what?

Now that we've went over the logistics of each, let's talk about when to use what. There are some obvious reasons you would use hashes over arrays, one being, the customizable key values for storing information, not in any particular order. On the other hand, if you want to make a list that needs to be ordered by order, well then, you'd use arrays.

The key here is knowing the number of elements. If you know the number of elements you're going to add to your list in advance, then you should use arrays. Arrays use up alot of memory, so you want to use them sparingly especially if you are creating a app intended for a mobile device. If you don't know, then you should use the Hashes, because of their flexibility.
